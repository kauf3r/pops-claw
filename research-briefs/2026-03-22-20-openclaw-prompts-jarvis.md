## TL;DR

Comprehensive workspace architecture for OpenClaw: 20 prompts covering SOUL.md personality, USER.md deep profile, HEARTBEAT.md batched checks, TOOLS.md local setup, LEARNINGS.md self-improvement loops, memory management, security rules, vibe coding standards, and multi-agent hierarchy. A full operating manual for making an agent actually useful.

## Key Insights

- SOUL.md should be a full character with voice and anti-slop standards, not just "be helpful"
- USER.md needs complete founder context (schedule, revenue sources, competitors) so agent never asks obvious questions
- Self-improvement loop: fix → write lesson → write rule → review at session start. Making the same mistake twice is unforgivable.
- Autonomy tiers: read/search/deploy freely, but spending money/deleting production/security needs approval
- "Mental notes don't survive sessions. Files do." — write everything down, promote important stuff from daily logs to MEMORY.md
