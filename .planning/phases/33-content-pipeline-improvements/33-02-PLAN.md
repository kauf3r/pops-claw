**1. "Write about X" command (CP-03):**
- When user says "write about X", "create an article about X", or similar
- Bob inserts a high-priority topic into content.db:
  ```sql
  sqlite3 /workspace/content.db "BEGIN IMMEDIATE;
  INSERT INTO topics (title, slug, brief, keywords, status, priority)
  VALUES ('<title>', '<slug-from-title>', '<brief from user context>', '<keywords>', 'backlog', 1);
  INSERT INTO pipeline_activity (entity_type, entity_id, agent_id, action, old_status, new_status, details)
  VALUES ('topics', last_insert_rowid(), 'main', 'created', NULL, 'backlog', 'On-demand request from user');
  COMMIT;"
  ```
- Priority 1 = highest, ensures Quill picks it up before lower-priority backlog items
- Bob confirms the topic was created and tells user it will be picked up by Quill at next writing-check cron (daily 11AM PT)
- If user says "trigger writing now" or "write it now", Bob can also mention that immediate triggering is available but note the cron will pick it up within 24h automatically

**2. "Research topics about X" command (CP-04):**
- When user says "research topics about X", "find article ideas about X", or similar
- Bob inserts a research directive note:
  ```sql
  sqlite3 /workspace/content.db "BEGIN IMMEDIATE;
  INSERT INTO topics (title, slug, brief, keywords, status, priority, category)
  VALUES ('Research: <topic area>', 'research-<slug>', 'User requested topic research about: <topic area>. Research trending angles, keywords, and content gaps.', '<keywords>', 'backlog', 1, 'research-directive');
  INSERT INTO pipeline_activity (entity_type, entity_id, agent_id, action, old_status, new_status, details)
  VALUES ('topics', last_insert_rowid(), 'main', 'created', NULL, 'backlog', 'Research directive from user');
  COMMIT;"
  ```
- Bob confirms the directive was created and tells user Vector will pick it up on next topic-research cron (Tue/Fri 10AM PT)

**3. "Get social posts for X" command (CP-05):**
- When user says "get social posts for <article>", "show me the LinkedIn post for <article>", or similar
- Bob queries content.db for social copy:
  ```sql
  sqlite3 -header -column /workspace/content.db "
  SELECT sp.platform, sp.content, sp.image_prompt, sp.status
  FROM social_posts sp
  JOIN articles a ON sp.article_id = a.id
  WHERE a.title LIKE '%<keyword>%'
  ORDER BY sp.platform;"
  ```
- Bob formats the results as ready-to-paste social copy, one section per platform (LinkedIn, Twitter/X, Instagram)
- If no social posts exist for the article, Bob explains that social copy is generated during the publish step by Ezra

Also include a quick reference section at the top with:
- Content DB path: `/workspace/content.db`
- Available statuses for topics: backlog, writing, review, approved, published, completed, done
- Available statuses for articles: draft, writing, review, revision, approved, published
- Cron schedule reference (when each agent runs)
- Channel IDs for content notifications

Use the established workspace protocol doc pattern (like VOICE_NOTES_PROTOCOL.md). The file should be comprehensive but scannable -- Bob needs to find the right SQL pattern quickly.

Transfer the file using base64 encoding to avoid shell escaping issues (per quick task 3 lessons learned).
