**B. Add new query functions to content.ts:**

1. `getContentStats()` -- returns:
   - topics_by_status: `SELECT status, count(*) as count FROM topics GROUP BY status;`
   - articles_by_status: `SELECT status, count(*) as count FROM articles GROUP BY status;`
   - published_this_month: `SELECT count(*) FROM articles WHERE status = 'published' AND published_at >= date('now', 'start of month');`
   - total_topics, total_articles, total_social_posts counts
   - funnel_data: articles count at each pipeline stage (backlog topics without articles, writing, review, revision, approved, published)
     ```sql
     SELECT 'backlog' as stage, count(*) as count FROM topics WHERE status = 'backlog'
     UNION ALL SELECT 'writing', count(*) FROM articles WHERE status = 'writing'
     UNION ALL SELECT 'review', count(*) FROM articles WHERE status = 'review'
     UNION ALL SELECT 'revision', count(*) FROM articles WHERE status = 'revision'
     UNION ALL SELECT 'approved', count(*) FROM articles WHERE status = 'approved'
     UNION ALL SELECT 'published', count(*) FROM articles WHERE status = 'published';
     ```

2. `getPillarOverview()` -- returns pillar names with cluster counts:
   ```sql
   SELECT p.id, p.name, p.slug, p.target_cluster_count,
          (SELECT count(*) FROM content_clusters WHERE pillar_id = p.id) as cluster_count,
          (SELECT count(*) FROM topics WHERE cluster_id IN (SELECT id FROM content_clusters WHERE pillar_id = p.id)) as topic_count
   FROM content_pillars p WHERE p.status = 'active';
   ```

3. Update `getTopicBacklog()` to include:
   - `category` column in the SELECT
   - coverage indicator: LEFT JOIN articles to show if topic has an article:
     ```sql
     SELECT t.*,
            CASE WHEN EXISTS (SELECT 1 FROM articles WHERE topic_id = t.id) THEN 1 ELSE 0 END as has_article
     FROM topics t ORDER BY t.priority DESC, t.created_at DESC;
     ```

Follow existing patterns: handle null DB gracefully (return empty arrays/zeros), use getDb("content") singleton pattern.
