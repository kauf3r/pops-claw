```markdown
# Growth Dashboard Protocol

Goals and journal entries live in the andyOS Dashboard at dashboard.andykaufman.net. Your role is the nudge layer — remind Andy to journal, surface goal summaries, and link to the dashboard for data entry.

## Evening Journal Nudge (8pm PT, via journal-nudge cron)

Send a brief Slack DM:
- Include today's prompt category (Monday=gratitude, Tuesday=challenges, etc.)
- Link to the journal page: dashboard.andykaufman.net/journal
- Example: "Evening reflection (gratitude): head to your journal when you're ready -> dashboard.andykaufman.net/journal"
- If Andy has already journaled today (check /api/growth/summary), say nothing.

## Morning Briefing Goals Section

When generating the morning briefing, fetch goal summary from the andyOS API:
```
curl -s -H "Authorization: Bearer $GROWTH_API_KEY" https://dashboard.andykaufman.net/api/growth/summary
```

Format the response as Section 13:
```
## Goals
- Get in shape (58%) — exercise 75%, weight 40%, pushups 60% [target: June 1]
- Read 12 books (25%) — books/month 33%, notes 17%
```
If no active goals or API unreachable, omit the section silently.

## Weekly Goal Check-In (Sunday 9am PT, via weekly-goal-checkin cron)

Send a Slack DM summarizing goal progress for the week:
- Fetch from /api/growth/summary
- For each active goal: title, overall progress %, each KR with current %
- Highlight goals with <30% progress or approaching target dates
- End with: "Review and update at dashboard.andykaufman.net/goals"
- If no active goals, send: "No active goals set. Start one at dashboard.andykaufman.net/goals"

## Goal Conversations via DM

If Andy mentions goals in DMs (create, update, check-in):
- Acknowledge the request
- Direct them to the dashboard: "I'll note that — update it at dashboard.andykaufman.net/goals for the full picture"
- Do NOT attempt to create or update goals yourself. The dashboard owns goal data.

## Journal Conversations via DM

If Andy sends what looks like a journal response via DM:
- Acknowledge it warmly
- Suggest saving it in the dashboard: "Great reflection. Drop it in your journal at dashboard.andykaufman.net/journal so it's tracked with mood/energy"
- Do NOT store journal entries yourself. The dashboard owns journal data.

## Important Rules

1. **Dashboard is the source of truth.** Never duplicate goal or journal data in local DBs.
2. **Nudge, don't nag.** One journal prompt per evening. No follow-up if ignored.
3. **Morning briefing only.** Don't mention goals outside the briefing unless asked.
4. **Weekly check-in on Sundays only.** Don't repeat goal progress mid-week unless asked.
5. **Link, don't replicate.** Always point to the dashboard URL for data entry.
```

2. **Create journal-nudge cron** (8pm PT = 3:00 UTC):

First check where crons live:
```
python3 -c "
import json, os
for p in ['/home/ubuntu/.openclaw/openclaw.json', '/home/ubuntu/.openclaw/cron/jobs.json']:
    if os.path.exists(p):
        with open(p) as f:
            d = json.load(f)
        crons = d if isinstance(d, list) else d.get('crons', [])
        print(f'{p}: {len(crons)} crons')
"
```

Then add the cron (adjust path based on findings):
```
/home/ubuntu/.npm-global/bin/openclaw crons create \
  --name "journal-nudge" \
  --schedule "0 3 * * *" \
  --agentId main \
  --text "Follow GROWTH_DASHBOARD.md Evening Journal Nudge section. Check if Andy has already journaled today via the growth summary API. If not, send a brief Slack DM with today's prompt category and a link to dashboard.andykaufman.net/journal. If already journaled, do nothing." \
  --delivery dm \
  --isolated true
```

If CLI fails, add directly to the cron JSON file (same pattern as Phase 55-03).
