1. Check if ~/clawd/research-briefs/ exists and has briefs:


2. If directory does not exist, create it:
```
mkdir -p ~/clawd/research-briefs
```

3. Check if already a git repo:
```
cd ~/clawd/research-briefs && git status 2>/dev/null || echo "NOT A GIT REPO"
```

4. If not a git repo, initialize:
```
cd ~/clawd/research-briefs && git init
```

5. Configure git identity (needed for commits from EC2):
```
cd ~/clawd/research-briefs && git config user.name "Bob (EC2)" && git config user.email "bob@pops-claw"
```

6. Add GitHub remote. The pops-claw repo uses HTTPS with a deploy key or SSH. Check if GitHub CLI or SSH key is available:
```
gh auth status 2>/dev/null || echo "NO GH CLI AUTH"
ls ~/.ssh/id_* 2>/dev/null || echo "NO SSH KEYS"
```

If `gh` is authenticated, use HTTPS remote:
```
cd ~/clawd/research-briefs && git remote add origin https://github.com/andykaufman/pops-claw.git 2>/dev/null || git remote set-url origin https://github.com/andykaufman/pops-claw.git
```

If no gh auth, check for SSH keys and use SSH remote:
```
cd ~/clawd/research-briefs && git remote add origin git@github.com:andykaufman/pops-claw.git 2>/dev/null || git remote set-url origin git@github.com:andykaufman/pops-claw.git
```

IMPORTANT: The research-briefs/ directory on EC2 is a SUBDIRECTORY of the pops-claw repo, not the repo root. Two approaches:
- Option A (preferred): Clone or init pops-claw repo on EC2, work within its research-briefs/ subdirectory
- Option B: Use a sparse checkout or subtree approach

Since the auto-brief skill needs to commit and push from inside EC2, the cleanest approach is:

A. Clone pops-claw repo to a dedicated location on EC2:
```
cd ~/clawd && git clone git@github.com:andykaufman/pops-claw.git pops-claw-repo 2>/dev/null || echo "ALREADY CLONED"
```
If SSH fails, try HTTPS with gh:
```
cd ~/clawd && gh repo clone andykaufman/pops-claw pops-claw-repo 2>/dev/null || echo "ALREADY CLONED"
```

B. Verify the clone has research-briefs/:
```
ls ~/clawd/pops-claw-repo/research-briefs/ | head -5
```

C. If existing briefs are in ~/clawd/research-briefs/ (not the repo), copy them to the repo:
```
cp ~/clawd/research-briefs/*.md ~/clawd/pops-claw-repo/research-briefs/ 2>/dev/null
```

D. Symlink ~/clawd/research-briefs to the repo subdirectory for backward compat:
```
if [ -d ~/clawd/research-briefs ] && [ ! -L ~/clawd/research-briefs ]; then
  mv ~/clawd/research-briefs ~/clawd/research-briefs.bak
fi
ln -sf ~/clawd/pops-claw-repo/research-briefs ~/clawd/research-briefs
```

E. Set git identity in the repo:
```
cd ~/clawd/pops-claw-repo && git config user.name "Bob (EC2)" && git config user.email "bob@pops-claw"
```

F. Test push capability:
```
cd ~/clawd/pops-claw-repo && git pull origin main && echo "PULL OK"
```

NOTE: If no git credentials exist on EC2, this task becomes a checkpoint:human-action. The executor should first try `gh auth status` and `ssh -T git@github.com` to check. If neither works, create a deploy key:
```
ssh-keygen -t ed25519 -f ~/.ssh/pops-claw-deploy -N ""
cat ~/.ssh/pops-claw-deploy.pub
```
Then pause and ask the user to add this as a deploy key on the GitHub repo (Settings > Deploy keys > Add, check "Allow write access").

After credential setup, configure SSH to use the deploy key for github.com:
```
echo -e "Host github.com\n  IdentityFile ~/.ssh/pops-claw-deploy\n  IdentityFIle ~/.ssh/id_ed25519" >> ~/.ssh/config
```
