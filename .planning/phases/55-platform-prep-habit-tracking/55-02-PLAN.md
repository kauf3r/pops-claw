SSH into EC2 and create /home/ubuntu/scripts/habit-manager.py with the following functionality.

The script operates on /workspace/growth.db (sandbox path) when run inside Docker, or accepts --db path for direct EC2 testing.

**Subcommands:**

1. `create <name> [--frequency daily|weekly] [--category health|mind|productivity|social|learning] [--description "..."]`
   - INSERT into habits table
   - Print confirmation with habit ID

2. `list [--status active|paused|archived] [--all]`
   - SELECT from habits, default status=active
   - For each habit, calculate and display:
     - Current streak (from streak_current column)
     - Best streak (from streak_best column)
     - Consistency rate: (total_completions / days_since_created) * 100, excluding paused days
     - Last logged date
   - Format as readable text output

3. `log <habit_name_or_id> [--status done|skipped] [--reason "..."] [--date YYYY-MM-DD]`
   - Default status=done, default date=today
   - INSERT OR IGNORE into habit_logs (UNIQUE constraint prevents double-logging)
   - Update streak logic:
     - If status=done:
       - If streak_last_date == yesterday OR streak_last_date == day-before-yesterday (1-day grace): increment streak_current, update streak_last_date
       - Else: set streak_current = 1, update streak_last_date
       - Update streak_best = max(streak_best, streak_current)
       - Increment total_completions
     - If status=skipped:
       - Store skip_reason in habit_logs
       - Do NOT break streak (skipped is an explicit log, not a miss)
       - Do NOT increment total_completions
   - Print confirmation with updated streak info

4. `pause <habit_name_or_id>`
   - UPDATE habits SET status='paused', paused_at=datetime('now')
   - Print confirmation

5. `archive <habit_name_or_id>`
   - UPDATE habits SET status='archived', archived_at=datetime('now')
   - Print confirmation

6. `status [--date YYYY-MM-DD]`
   - Show all active habits with today's log status (done/skipped/not yet logged)
   - Show streak and consistency for each
   - This is what the morning briefing and evening nudge will call

7. `unlogged [--date YYYY-MM-DD]`
   - List active habits NOT logged for the given date (default today)
   - Used by evening recap nudge

**Implementation notes:**
- Use argparse for CLI parsing
- Use sqlite3 module (available in Docker sandbox)
- DB path: default `/workspace/growth.db`, override with `--db` flag for testing
- Habit name matching: case-insensitive partial match (e.g., "med" matches "meditation")
- All dates in YYYY-MM-DD format, all timestamps in ISO 8601
- Output as plain text (not JSON) -- Bob will read and relay to user
- Include shebang: `#!/usr/bin/env python3`
- Make executable: `chmod +x /home/ubuntu/scripts/habit-manager.py`

**Triggers:** "pause", "hold", "take a break from"

## Streak Rules

- **1-day forgiveness:** Missing a single day does NOT break the streak. Missing 2 consecutive days resets to 0.
- **Skips are not misses:** An explicitly logged skip ("skipped gym -- sick") preserves the streak.
- **Consistency rate** is the primary metric, shown as percentage: completions / eligible days.
- **Streaks** are the secondary metric, shown alongside consistency.
- Never say "streak broken" or "you failed." Frame as "starting a new streak" or show consistency rate.

## Morning Briefing Integration

When generating the morning briefing (Section 12), run:
```
python3 /workspace/habit-manager.py status
```
Format as:
```
## Habits
- meditation: 12-day streak, 94% consistency
- exercise: 3-day streak, 60% consistency (due today)
- reading: not yet logged today, 45% consistency
```

## Evening Recap Integration

When generating the evening recap, run:
```
python3 /workspace/habit-manager.py unlogged
```
If there are unlogged habits, include a gentle nudge:
"You haven't logged **meditation** or **gym** today. Done, skipped, or skip for today?"

If all habits are logged: "All habits logged today -- nice work."
If no active habits exist: omit the section entirely.

## Important Rules

1. **Max 4 proactive DMs per day** -- self-improvement messages count toward this cap.
2. **Never nag.** One nudge in the evening recap is enough. If Andy doesn't respond, do not follow up.
3. **Celebrate weekly, not daily.** Daily acknowledgment: "Logged." Weekly milestone: "86% consistency this week, up from 71%."
4. **Pause without guilt.** If Andy says "pause" or "not now," comply immediately without "are you sure?"
5. **Silence is fine.** If Andy ignores 3+ consecutive nudges, mention once: "I've noticed you haven't been logging habits. Want me to pause nudges for a bit?" Then respect the answer.
