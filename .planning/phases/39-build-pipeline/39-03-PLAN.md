    The payload currently says: "Read /workspace/yolo-dev/YOLO_BUILD.md and follow its instructions to execute tonight's autonomous build."

    Rewrite the cron payload to be an explicit, action-forcing instruction. Instead of "Read X and follow its instructions," make the payload itself the first step with explicit tool-use language:

    New payload (adjust based on Step 1 findings):
    ```
    AUTONOMOUS BUILD TASK — Execute immediately. Open and read /workspace/yolo-dev/YOLO_BUILD.md. Follow every step in that document sequentially. Start with Step 0 (Pre-Flight Checks). You MUST use the bash tool and python tool to execute every code block. Do not summarize or acknowledge — ACT. Begin now.
    ```

    The key insight: cron payloads need to be imperative ("do this NOW") rather than suggestive ("read this and follow it"). The word "ACT" and explicit tool mentions may nudge the agent into agentic mode rather than conversational mode.

    **Prong B — Add maxTurns to the cron entry if supported:**

    If `openclaw cron add --help` shows a `--max-turns` or `maxTurns` field, add `maxTurns: 15` to the cron entry.

    If NOT supported in the cron schema, this is handled by Task 2 (adding turn-budget to YOLO_BUILD.md).
