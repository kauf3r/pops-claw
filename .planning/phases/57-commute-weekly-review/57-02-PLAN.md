### Habit Summary
Query growth.db for this week's habit completions:
```python
import sqlite3
conn = sqlite3.connect('/workspace/db/growth.db')
c = conn.cursor()

# Active habits
habits = c.execute("SELECT id, name, streak_current, streak_best, total_completions FROM habits WHERE status = 'active'").fetchall()

# This week's logs (Mon-Sun, using date('now', 'weekday 0', '-6 days') for Monday)
week_start = c.execute("SELECT date('now', '-7 hours', 'weekday 0', '-6 days')").fetchone()[0]
logs = c.execute("""
    SELECT h.name, hl.date, hl.status
    FROM habit_logs hl
    JOIN habits h ON h.id = hl.habit_id
    WHERE hl.date >= ?
    ORDER BY hl.date
""", (week_start,)).fetchall()

conn.close()
```

Format as:
- Each active habit: name, days completed this week / days expected, streak status
- Example: "Meditate: 5/7 days (71%), 12-day streak"
- If no habits exist: "No active habits yet."
- If habits exist but 0 logs: "Active habits: [names]. No completions logged this week."

### Goal Progress
Fetch goal summary from the andyOS Dashboard API:

Format the response:

- Example: "Get in shape (58%, target Jun 1) -- exercise 75%, weight 40%, pushups 60%"
- If no goals or API unreachable: "No active goals. Set one at dashboard.andykaufman.net/goals"

### Energy Correlations
Correlate Oura health data with habit completion and mood for the week. Query health.db:
```python
import sqlite3

health = sqlite3.connect('/workspace/health.db')
week_start_row = health.execute("SELECT date('now', '-7 hours', 'weekday 0', '-6 days')").fetchone()

snapshots = health.execute("""
    SELECT date, sleep_score, readiness_score, hrv_balance, resting_hr, activity_score

health.close()

growth = sqlite3.connect('/workspace/db/growth.db')
habit_days = growth.execute("""
```

Format:
- Week averages: sleep score, readiness score, HRV, resting HR

- If <3 days of Oura data: show what's available, skip correlation

### Commute Prompt Recap
Query growth.db for this week's commute prompts and responses:
```python
import sqlite3
conn = sqlite3.connect('/workspace/db/growth.db')
week_start = conn.execute("SELECT date('now', '-7 hours', 'weekday 0', '-6 days')").fetchone()[0]
prompts = conn.execute("""
    SELECT date, prompt_category, substr(prompt_text, 1, 120),
           CASE WHEN response_text IS NOT NULL THEN 'responded' ELSE 'unanswered' END,
           key_insights
    FROM commute_prompts
    WHERE date >= ?
    ORDER BY date
""", (week_start,)).fetchall()
conn.close()
```

Format:
- Prompts sent: X/5 weekdays, Y responded
- If responses exist, highlight notable themes from the key_insights column
- If no prompts this week: "No commute prompts delivered this week."

### Reflection Prompt
Based on all the patterns above, generate ONE thought-provoking question for the upcoming week. Guidelines:
- If habits are strong: push toward a growth edge ("What's one thing you've been avoiding?")
- If sleep/readiness is low: orient toward recovery ("What would a restful week look like?")
- If goals are stalled: gentle accountability ("Which goal would feel best to make progress on?")
- If everything looks great: celebration + next level ("What made this week work? How do you build on it?")
- Keep it to 1-2 sentences. This sets the tone for the coming week.

```markdown
## Weekly Growth Review (via weekly-review cron, Sunday 8am PT)

The existing weekly-review cron includes a Growth Review section after the standard health/calendar/YOLO content. This section covers:

1. **Habit Summary** -- completion rates, streaks from growth.db
2. **Goal Progress** -- fetched from andyOS API (/api/growth/summary with $GROWTH_API_KEY)
3. **Journal & Mood Trends** -- from andyOS API + growth.db journal_entries
4. **Energy Correlations** -- cross-reference Oura sleep/readiness (health.db) with habit completion days
5. **Commute Prompt Recap** -- this week's prompts and response rate from growth.db
6. **Reflection Prompt** -- one question for next week based on patterns
7. **Store Review** -- INSERT into growth.db weekly_reviews table

### Graceful Degradation

Each section handles missing data independently. Priority:
- Oura data is most reliable (62+ days history). Always include health trends.
- Habits/journals may be sparse early on. Show what exists, invite usage if empty.
- Goals API may be unreachable. Skip gracefully, don't error.
- Commute prompts may have 0 entries if the cron just started. Say so and move on.

### Storage Format (growth.db weekly_reviews)

| Column | Content |
|--------|---------|
| week_start | Monday of the review week (YYYY-MM-DD) |
| went_well | Prose: wins, completed habits, goal progress |
| to_improve | Prose: missed habits, stalled goals, low-energy days |
| insights | Prose: correlation findings + reflection prompt |
| oura_summary | Compact: "sleep:82/readiness:78/hrv:45/rhr:58" |
| habit_summary | Compact: "meditate:5/7,exercise:3/7" |
| goal_summary | Compact: "Get in shape:58%,Read 12 books:25%" |

### Rules

1. **Don't duplicate the weekly-goal-checkin cron.** That runs at 9am PT with goal-specific detail. The growth review's goal section is a brief summary, not a full check-in.
2. **Correlation requires both data sources.** Don't claim "sleep helped habits" unless there are both Oura rows AND habit logs for the same days.
3. **One review per week.** INSERT OR REPLACE on week_start prevents duplicates if the cron retries.
4. **Keep the DM under 40 lines total** (including existing weekly review content). Be concise.
```

## Risks & Mitigations

- **Payload size:** Weekly-review payload grows from ~3KB to ~6KB. Well within OpenClaw limits (payloads tested up to 10KB in Phase 41). If payload is too large, consolidate the Python query examples into a single block.
- **health.db sandbox path:** The workspace maps `~/clawd/agents/main/` to `/workspace/`, so health.db is at `/workspace/health.db`. growth.db is at `/workspace/db/growth.db` (bind-mounted separately). If path errors occur, check `ls -la /workspace/*.db /workspace/db/*.db` from Bob's sandbox.
- **Empty data graceful degradation:** All sections have explicit "if no rows" fallback text. The cron payload mandates "never error out" per the same pattern as 57-01 commute prompts.
- **andyOS API down:** Goal and journal sections skip cleanly. The curl call includes `$GROWTH_API_KEY` which is already configured in sandbox env from Phase 56-03.
- **Duplicate weekly reviews:** `INSERT OR REPLACE` on `week_start` UNIQUE constraint prevents duplicates if the cron fires twice or is manually triggered after the automatic run.
- **Correlation with sparse habit data:** 0 habit logs means correlation section shows Oura trends only and notes "No habit data to correlate yet." This is expected for the first few weeks.
