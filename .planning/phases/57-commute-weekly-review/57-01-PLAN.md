# 57-01: Morning Commute Prompt System

**Goal:** Bob sends a personalized, context-rich reflection question via Slack DM at 7:15am PT weekdays, stores it in growth.db, and links voice note responses back to the prompt.
**Estimated plans within:** 1

## Tasks

### Task 1: Create commute-prompt cron

**What:** New OpenClaw cron targeting `main` agent with isolated session. Weekdays only, 7:15am PT = 14:15 UTC. The cron payload is a self-contained prompt telling Bob exactly what to do: gather context, generate prompt, store in growth.db, deliver via Slack DM.

**Where:** EC2 via SSH (`ssh -i ~/.ssh/clawdbot-key.pem ubuntu@100.72.143.9`)

**Commands:**

```bash
/home/ubuntu/.npm-global/bin/openclaw cron create \
  --name "commute-prompt" \
  --schedule "15 14 * * 1-5" \
  --agent main \
  --message "Morning commute prompt delivery. Follow these steps exactly:

STEP 1 - GATHER CONTEXT:
a) Calendar: Run gog calendar events --account=theandykaufman@gmail.com --all --today --max=10 to get today's meetings.
b) Oura sleep: Query health.db for last night's sleep data:
   sqlite3 /workspace/db/health.db \"SELECT sleep_score, readiness_score, hrv_balance, resting_hr FROM health_snapshots WHERE date = date('now', '-7 hours') ORDER BY date DESC LIMIT 1\"
   (The -7 hours adjusts UTC to PT for 'last night')
c) Goals: Fetch active goals from andyOS API:
   curl -s -H 'Authorization: Bearer \$GROWTH_API_KEY' https://dashboard.andykaufman.net/api/growth/summary
d) Recent commute responses: Check last 3 answered prompts for variety:
   sqlite3 /workspace/db/growth.db \"SELECT prompt_category, substr(prompt_text, 1, 80) FROM commute_prompts WHERE response_text IS NOT NULL ORDER BY date DESC LIMIT 3\"

STEP 2 - GENERATE PROMPT:
Using the context gathered, craft ONE personalized reflection question for Andy's commute. Guidelines:
- Rotate categories: reflection, planning, gratitude, creative, growth. Avoid repeating the same category as the last 2 prompts.
- Reference specific context: mention a meeting name, sleep score, or active goal when relevant.
- Keep it to 1-3 sentences. End with an invitation to record a voice note.
- If sleep score < 70 or readiness < 70, lean toward gentler/recovery-focused prompts.
- If a big meeting is today, lean toward preparation/intention-setting prompts.
- If no calendar or Oura data available, use goals or a general growth prompt. Never error out.

STEP 3 - STORE IN GROWTH.DB:
Pick the category you chose and store:
sqlite3 /workspace/db/growth.db \"INSERT OR REPLACE INTO commute_prompts (date, prompt_text, prompt_category, delivered_at) VALUES (date('now', '-7 hours'), '<your prompt text>', '<category>', datetime('now'))\"

STEP 4 - DELIVER:
Send the prompt as a Slack DM to Andy. Format:
'Good morning! Here is your commute reflection:

<prompt text>

Record a voice note on your drive, or reply here anytime today.'

Do NOT include raw data dumps. The DM should feel like a thoughtful friend, not a data report." \
  --no-deliver \
  --lightContext
```

**Verify:**
```bash
ssh -i ~/.ssh/clawdbot-key.pem ubuntu@100.72.143.9 \
  '/home/ubuntu/.npm-global/bin/openclaw cron list' | grep commute-prompt
# Should show: commute-prompt, schedule 15 14 * * 1-5, agent main
```

**Notes:**
- `--no-deliver` = isolated session, Bob sends DM himself (same pattern as journal-nudge)
- `--lightContext` = minimize token usage (same pattern as all heartbeat/nudge crons)
- `15 14 * * 1-5` = 7:15am PT on Mon-Fri (UTC offset: PT = UTC-7 during PDT)
- CLI uses `--agent` not `--agentId`, `--message` not `--text` (learned from Phase 56-03)

### Task 2: Add Morning Commute section to GROWTH_COMPANION.md

**What:** Append a "Morning Commute Prompts" section to the existing GROWTH_COMPANION.md protocol doc. This tells Bob how to handle voice note responses to commute prompts and sets behavioral rules.

**Where:** `/home/ubuntu/clawd/agents/main/GROWTH_COMPANION.md` on EC2

**Commands:** SSH in and append this section to the end of GROWTH_COMPANION.md:

```markdown
## Morning Commute Prompts

### Delivery (via commute-prompt cron, 7:15am PT weekdays)

The commute-prompt cron handles delivery. You do not need to send commute prompts outside of the cron. If Andy asks for today's commute prompt in a DM, check growth.db:
```
sqlite3 /workspace/db/growth.db "SELECT prompt_text FROM commute_prompts WHERE date = date('now', '-7 hours')"
```
If one exists, share it. If not, generate one on the spot following the cron's prompt generation logic.

### Voice Note Response Linking

After the voice-notes-processor transcribes a new voice note, check if today has an unresponded commute prompt:

```
sqlite3 /workspace/db/growth.db "SELECT id, prompt_text FROM commute_prompts WHERE date = date('now', '-7 hours') AND response_text IS NULL LIMIT 1"
```

If a match exists AND the voice note was recorded today (check coordination.db voice_notes.created_at), link them:

```
sqlite3 /workspace/db/growth.db "UPDATE commute_prompts SET response_text = '<transcription>', response_source = 'voice', responded_at = datetime('now') WHERE date = date('now', '-7 hours') AND response_text IS NULL"
```

Do NOT overwrite an existing response. If response_text is already set, the voice note is unrelated to the commute prompt -- just store it normally in coordination.db.

### Insight Extraction

After linking a voice note response (or a DM response) to a commute prompt, extract 2-3 key insights or themes from the response text. Store them:

```
sqlite3 /workspace/db/growth.db "UPDATE commute_prompts SET key_insights = '<2-3 bullet points summarizing key themes>' WHERE date = date('now', '-7 hours') AND response_text IS NOT NULL AND key_insights IS NULL"
```

Guidelines for insight extraction:
- Identify the top 2-3 themes, decisions, or realizations in the response
- Keep each insight to one sentence
- Format as a semicolon-separated list (e.g., "Feeling energized about Q2 planning; Need to prioritize sleep this week; Grateful for team support")
- If the response is too short or vague for meaningful insights, set key_insights to "Brief response -- no actionable insights extracted"

### DM Response Linking

If Andy replies in Slack DM and today has an unresponded commute prompt, treat the DM as the response:

```
sqlite3 /workspace/db/growth.db "UPDATE commute_prompts SET response_text = '<dm text>', response_source = 'dm', responded_at = datetime('now') WHERE date = date('now', '-7 hours') AND response_text IS NULL"
```

After storing, extract insights using the same process described above.

### Prompt Categories & Rotation

Categories: reflection, planning, gratitude, creative, growth

Check the last 2 delivered prompts to avoid repeating:
```
sqlite3 /workspace/db/growth.db "SELECT prompt_category FROM commute_prompts ORDER BY date DESC LIMIT 2"
```

### Settings

- **Commute prompt time:** 7:15am PT (cron schedule)
- **Days:** Weekdays only (Mon-Fri)
- **Response window:** Same calendar day (date-level matching, not time-based)
- **Max 1 prompt per day** (UNIQUE constraint on date column)

### Important Rules

1. **One prompt per day.** Never send a second commute prompt, even if asked.
2. **Graceful with missing data.** If Oura, calendar, or goals API is unavailable, still send a prompt using whatever context IS available.
3. **Warm, not clinical.** The prompt should sound like a thoughtful friend, not a data dashboard.
4. **Don't follow up.** If Andy doesn't respond, that's fine. No "Did you get a chance to reflect?" messages.
5. **Voice note linking is best-effort.** The 2-hour pipeline delay means the link happens later in the day. That's expected.
6. **Always extract insights after linking.** Every response (voice or DM) should get key_insights populated.
```

## Risks & Mitigations

- **PDT/PST shift:** `15 14 * * 1-5` is correct for PDT (Apr-Nov). When clocks fall back to PST (UTC-8), this fires at 6:15am PT. Mitigation: adjust cron in November, or accept 6:15am as still fine.
- **Oura data lag:** health.db syncs daily but may not have "last night" data by 7:15am. Mitigation: prompt generation gracefully falls back to goals/calendar if Oura row is missing.
- **andyOS API unreachable:** Goals endpoint could be down or GROWTH_API_KEY invalid. Mitigation: cron payload says "never error out" -- use whatever context IS available.
- **Voice note pipeline delay:** Voice notes recorded at 7:30am won't be transcribed until 8:00 or 10:00am cron run. Mitigation: documented as expected behavior. Linking happens asynchronously.
- **CLI flag changes on gateway update:** OpenClaw CLI flags may change between versions. Mitigation: if `cron create` fails, fall back to editing `/home/ubuntu/.openclaw/cron/jobs.json` directly (same fallback used in Phase 55-03).
- **ALTER TABLE idempotency:** If `key_insights` column already exists, SQLite will error "duplicate column name." This is harmless -- check with `.schema` first or ignore the error.
- **Voice-notes-processor payload size:** Appending ~1KB of commute linking instructions to the existing payload. Well within OpenClaw limits (tested up to 10KB in Phase 41).
