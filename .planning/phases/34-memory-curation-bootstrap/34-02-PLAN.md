# Plan 34-02: Fix Memory Flush Consistency

<objective>
Make daily log entries appear for every active day by fixing the flush trigger gap — short cron sessions never hit compaction, so the memoryFlush prompt never fires.

Purpose: Bob should have a complete daily log trail, not just from occasional long sessions.
</objective>

<context>
- Flush config: agents.defaults.compaction.memoryFlush.enabled=true, softThresholdTokens=6000
- Problem: Cron sessions (heartbeats, briefings) are typically under 6000 tokens, so compaction never triggers, so flush never fires
- Daily logs: Only 9 entries in 3 weeks (Feb 2-22), missing most days
- Flush prompt: "Write any lasting notes to memory/$(date +%Y-%m-%d).md"
- Memory dir: ~/clawd/agents/main/memory/
</context>

## Wave 1

### Task 1: Diagnose Flush Gap

SSH into EC2. Correlate daily log dates with session token counts to confirm the hypothesis:

```sql
-- Which sessions were long enough to trigger compaction?
SELECT date(created_at) as day, sum(input_tokens) as total_input, count(*) as calls
FROM llm_calls
WHERE agent_id = 'main'
GROUP BY date(created_at)
ORDER BY day DESC
LIMIT 20;
```

Cross-reference with existing daily log filenames to confirm: logs only appear on days with high-token sessions.

### Task 2: Add Daily Summary Cron

Create a lightweight evening cron (11 PM PT) that runs on main agent with Sonnet. Its only job is to review the day and write a daily log entry. Because it's a real conversation (not a quick heartbeat), it should hit compaction or at least generate enough context for the flush to trigger.

Cron config:
- ID: `daily-memory-flush`
- Schedule: `0 7 * * *` (7 AM UTC = 11 PM PT)
- Agent: main
- Model: sonnet
- Isolated: true
- Delivery: none (silent)
- Prompt: Reference doc pattern — create DAILY_FLUSH.md in workspace

DAILY_FLUSH.md instruction:
```
Review today's activity. Check coordination.db for completed tasks, email.db for conversations,
content.db for pipeline progress. Write a concise daily log entry to memory/YYYY-MM-DD.md with:
- Key events and decisions
- Anything you learned or should remember
- Open items carried to tomorrow
If nothing notable happened, write a one-line "quiet day" entry. Always write something.
```

### Task 3: Lower softThresholdTokens

Reduce from 6000 to 3000. This makes flush trigger earlier in sessions, catching more mid-length conversations. Risk is minimal — worst case is more frequent (small) memory writes.

```json
"memoryFlush": {
  "enabled": true,
  "softThresholdTokens": 3000,
  ...
}
```

### Task 4: Verify Over 24-48 Hours

After deploying cron + config change:
1. Check that daily-memory-flush cron appears in `openclaw cron list`
2. Wait for next natural cycle
3. Verify new daily log file appears in ~/clawd/agents/main/memory/
4. Check `openclaw memory status` shows incremented file/chunk count
