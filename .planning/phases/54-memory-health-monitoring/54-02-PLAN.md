# Plan 54-02: Set Up Health Check Cron and Alerting

## Objective
Schedule daily health check and configure Slack DM alert on failure.

## Tasks

<task id="1" name="Add crontab entry">
Add to ubuntu's crontab:
0 8 * * * /home/ubuntu/scripts/memory-health-check.sh >> /home/ubuntu/scripts/memory-health-check.log 2>&1
(08:00 UTC = 1am PT, after overnight QMD indexing)
</task>

<task id="2" name="Add Slack alert on failure" depends="1">
Modify health check script to send Slack DM via openclaw session on failure.
Use: openclaw sessions send --channel=dm --message="Memory health check FAILED: {details}"
Or use a simpler approach: write failure to a file that Bob checks in next heartbeat.
</task>

<task id="3" name="Verify cron is scheduled" depends="1">
Check crontab -l shows the entry.
Run script manually to confirm alert mechanism works.
</task>

## Verification
- [ ] Health check cron runs daily at 08:00 UTC
- [ ] Failure triggers Slack DM alert
- [ ] No false positives in initial run

## must_haves
- Cron runs daily and alerts on failure via Slack DM
