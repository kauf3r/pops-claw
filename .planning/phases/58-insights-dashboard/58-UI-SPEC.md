## Design System

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap within card headers (`gap-1`) |
| sm | 8px | Compact element spacing, card content `space-y-1` |
| md | 16px | Default element spacing, card grid `gap-4`, page `p-6` |
| lg | 24px | Page section spacing (`space-y-6`) |
| xl | 32px | Not used this phase |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: Sparkline chart height at 40px (`h-10`), matching existing OuraSparkline pattern.

### Sparkline Pattern (reuse existing)

All sparklines use the existing `OuraSparkline` component pattern:
- Container: `h-10 w-full`
- Chart: `ResponsiveContainer` 100% width/height
- Gradient: `linearGradient` from `hsl(var(--primary))` opacity 0.3 to 0
- Stroke: `hsl(var(--primary))` at 1.5px width
- No dots, no animation, connectNulls enabled
- Margins: `{ top: 2, right: 0, left: 0, bottom: 0 }`

For domain-specific sparklines (habits, mood, etc.), replace `hsl(var(--primary))` with the appropriate `var(--chart-N)` variable.

### Loading States

- Card with `h-full`

Full-width Insights card loading:
- Same card skeleton but with `col-span-full`
- Additional `Skeleton h-3 w-48` lines for correlation/theme text

Skeleton animation delay stagger: 75ms increments per card (matching overview/loading.tsx pattern).

## Interaction Contracts

### Card Hover

Cards with links: `hover:bg-accent/50 transition-colors cursor-pointer` (matching all existing hub cards).

Habits card and Insights card: No hover effect (no link target). Use `cursor-default`.

### Data Loading

- Server components with async data fetching (RSC pattern)
- Each card wrapped in `<Suspense fallback={<HubCardSkeleton />}>`
- Data fetched from local PostgreSQL (no cross-network calls per D-08)
- No SWR client-side refresh needed — server components re-render on navigation

### Responsive Breakpoints

| Breakpoint | Grid Layout |
|------------|-------------|
| Default (mobile) | 1 column, all cards stacked |
| `sm` (640px) | 2 columns for top row, insights full-width |
| `lg` (1024px) | 4 columns for top row, insights full-width |

### Sidebar Navigation

Add `/growth` to the LIFE section in `dashboard-shell.tsx`:

```typescript
// In navSections LIFE items array, after Journal:
{ label: "Growth", href: "/growth", icon: Sprout }
```

Icon: `Sprout` from lucide-react (represents growth/progress). Add to both `navSections` and `moreSheetSections` LIFE arrays.

### Theme Badges

Use `<Badge variant="secondary">` for journal theme tags. Each badge shows: `{category}: {theme text}`. Example: "recurring: work-life balance".
