## Memory Retrieval Protocol

Before responding to topics in these categories, ALWAYS search memory first:

### Trigger Categories
1. **User Preferences** — communication style, briefing format, tool choices
   - Example: `qmd search "Andy prefers"` or `qmd search "briefing format"`
2. **Project History** — past decisions, milestone context, what was tried before
   - Example: `qmd search "email integration decision"` or `qmd search "v2.7 changes"`
3. **System Configuration** — current versions, paths, credentials, cron schedules
   - Example: `qmd search "gateway config"` or `qmd search "cron schedule"`
4. **AirSpace Integration** — vendor context, meeting history, project status
   - Example: `qmd search "AirSpace security"` or `qmd search "Safe and Sound"`

### Consequence
If you're unsure about a user preference, system detail, or past decision — search memory BEFORE guessing. Getting it wrong and having to be corrected wastes more time than a 2-second search.

### How to Search
```bash
qmd search "your query here"
```
Results come from MEMORY.md and daily logs. If no results, state that clearly rather than making assumptions.
