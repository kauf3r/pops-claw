SSH to EC2 (100.72.143.9) as ubuntu.

1. Create the yolo-dev directory (no sudo):
   ```
   mkdir -p ~/clawd/yolo-dev
   ```

2. Create yolo.db with the builds table schema using the host sqlite3 binary (NOT the sandbox sqlite3-compat, which has version mismatch issues). Use `tee` to pipe the SQL, not heredoc (heredocs over SSH can hang with leading spaces):

   ```sql
   CREATE TABLE IF NOT EXISTS builds (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       date TEXT NOT NULL,
       slug TEXT NOT NULL,
       name TEXT NOT NULL,
       description TEXT,
       status TEXT NOT NULL DEFAULT 'idea' CHECK(status IN ('idea', 'building', 'testing', 'success', 'partial', 'failed')),
       tech_stack TEXT,
       lines_of_code INTEGER,
       files_created INTEGER,
       self_score INTEGER CHECK(self_score BETWEEN 1 AND 5),
       self_evaluation TEXT,
       build_log TEXT,
       error_log TEXT,
       started_at TEXT,
       completed_at TEXT,
       duration_seconds INTEGER,
       created_at TEXT NOT NULL DEFAULT (datetime('now'))
   );

   CREATE INDEX idx_builds_status_date ON builds(status, date);
   ```

3. Verify:
   - `sqlite3 ~/clawd/yolo-dev/yolo.db ".schema builds"` shows all columns
   - `sqlite3 ~/clawd/yolo-dev/yolo.db ".tables"` shows "builds"
   - `ls -la ~/clawd/yolo-dev/yolo.db` shows ubuntu:ubuntu ownership, ~12KB size

4. Test a write/read/delete cycle:
   ```
   sqlite3 ~/clawd/yolo-dev/yolo.db "INSERT INTO builds (date, slug, name, status) VALUES ('2026-02-24', 'infra-test', 'Infrastructure Test', 'success'); SELECT id, slug, status FROM builds; DELETE FROM builds WHERE slug='infra-test';"
   ```
   Should insert, display row, and delete cleanly. Confirms CHECK constraints and AUTOINCREMENT work.
